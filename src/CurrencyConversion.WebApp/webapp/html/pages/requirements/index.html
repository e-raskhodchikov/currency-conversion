Currency Conversion Business Requirements

  1. Accessible in a web browser
  2. User interface musts
    • accept money amount and currency type to convert from
    • accept currency type to convert to
    • invoke the conversion
    • display the converted money amount
    • notify the user nicely if something went wrong
  3. Exchange rates will be delivered daily to the app with the following exchange rates data
    • currency from
    • currency to
    • exchange rate
    • date
  4. Exchange rates will be stored in a local database
  5. Exchange rates feed is known to not deliver rates on a given day. Notify the end user of the date of the rate used
  6. Currency types may become invalid and the app will need to handle this rare situation (i.e. European Union breaks up and each country reverts back to their own currency)
